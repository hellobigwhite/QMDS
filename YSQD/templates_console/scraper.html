域名直爬

直接粘贴 Shopify 域名，系统会自动导入并爬取商品数据到 shopify_data_new，相当于"导入 URL + 商品爬取"一步完成。

查看任务中心

Shopify URL 自动分类

系统会自动判断 Shopify URL 属于哪个大类。高置信度结果会移动到对应的 *_Filtered_URLs 集合，低置信度或非 Shopify URL 会从来源集合删除。

建议默认值：最大处理数 100，最低命中分 12，最小领先分差 4。分数越高越严格，分差越大越能避免相近类目误判。

查看任务中心

商品数据爬取

从 `shopify_url` 的 `_Filtered_URLs` 或 `_Unfiltered_URLs` 读取来源，先把当前任务按分类写入 `_Reuse_Queue`，再按时间和次数规则把到期旧 URL 回投到任务池，最后爬取商品数据到 `shopify_data_new`。

填 0 表示不限制

单类目集合商品爬取

只从选定的单个 MongoDB 集合（如 五金_Filtered_URLs）中读取来源 URL，不遍历所有类目，适用于定向补爬某个类目的商品数据。选好集合后点击下方按钮，系统会仅从该集合加载待爬 URL。

查看任务中心

导航分类商品爬取

解析店铺导航栏获取两级分类，按集合逐类爬取商品。商品会使用导航栏的两级分类名作为 source_category 和 source_subcategory。

说明：此模式会先访问店铺首页解析导航栏菜单，按导航中的集合分类逐页爬取商品。 商品保存时 source_category = 导航一级分类名，source_subcategory = 导航二级分类名。 若导航解析失败，会自动回退到 collections.json 用集合标题推断两级分类。
查看任务中心

复用池回投到任务池

把 `_Reuse_Queue` 里已经到期、且满足次数规则的旧 URL 重新投放回任务池，用于补充后续爬取任务。

查看任务中心

商品数据导出

只导出你选中的一个集合，并在集合内部按商品的 `分类` 做小分类限制。小分类不足最小值会跳过，总量不够时优先保留商品数量更多的小分类，分类内再按标题质量优先导出。

导出成功后会先备份到 `shopify_data_backup`，再从源集合删除本次导出的商品。
查看任务中心

数据库直接导出商品

直接导出 `shopify_data_new` 数据库下的 `_clean` 集合。导出成功后会先备份到 `shopify_data_backup`，再从源集合删除本次导出的商品。

填 0 表示导出全部。
