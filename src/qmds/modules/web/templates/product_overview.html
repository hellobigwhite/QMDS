数据库结构说明

URL来源数据库
qmds_url_stores - 存储店铺URL数据
URL来源集合
{category}_filtered - 存储已筛选的店铺URL
产品数据数据库
qmds_product_data - 存储商品数据
原始数据集合
{category}_raw - 存储爬取的原始商品数据
清洗后数据集合
{category}_clean - 存储清洗后的商品数据
类目对应关系
同一类目的原始数据和清洗后数据通过类目名称对应，如 electronics_raw 和 electronics_clean
数据爬取流程
从 qmds_url_stores.{category}_filtered 获取URL → 爬取 /products.json → 保存到 qmds_product_data.{category}_raw
数据清洗流程
从 {category}_raw 读取数据 → 过滤无效数据、违禁词处理 → 保存到 {category}_clean
